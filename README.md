🚀 Step-by-Step GitHub Pages Deployment Guide

Use these steps to upload your file to GitHub and run it directly on your Android mobile device like a native application:

Step 1: Create a Repository on GitHub

Sign in to your GitHub account.

Click the green "New" button to create a new repository.

Set the repository name to za-pc-matrix.

Choose Public (required for free GitHub Pages).

Click "Create repository".

Step 2: Upload index.html & README.md

On your repository's welcome page, click the link that says "uploading an existing file".

Drag and drop both the index.html and README.md files into the box.

Scroll down and click the green "Commit changes" button.

Step 3: Turn on GitHub Pages Hosting

Go to your repository's Settings tab (the gear icon on the top horizontal bar).

On the left sidebar menu, scroll down to the Code and automation section and click Pages.

Under Build and deployment -> Branch, click the dropdown and change it from None to main (or master).

Click Save.

Wait 1 to 2 minutes for GitHub to build the files. Refresh the page, and you will see your public live link (e.g., https://yourusername.github.io/za-pc-matrix/).

📱 How to Run as an Android App

Open your live GitHub Pages link inside Google Chrome on your Android phone.

Tap the three vertical dots (Menu) in the top right-hand corner of Chrome.

Tap "Add to Home screen".

Give your shortcut a clean name like PC Matrix and hit add.

Tap the new icon on your phone’s home screen to launch it in a sleek, full-screen standalone application layout with local memory storage active.
